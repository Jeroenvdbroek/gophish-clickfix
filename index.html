Achtergrond

Wat is een ClickFix aanval?

ClickFix — ook bekend als FakeCaptcha — is een van de gevaarlijkste social engineering technieken van dit moment. Niet omdat het technisch geavanceerd is, maar juist omdat het zo simpel werkt.

Bij een ClickFix aanval maakt een aanvaller een neppe verificatiepagina die er volledig legitiem uitziet — denk aan een Microsoft MFA setup scherm, een Cloudflare CAPTCHA, of een browser verificatieprompt. De pagina vraagt de gebruiker om een "verificatiestap" uit te voeren.

Wat de gebruiker niet weet: op het moment dat ze op de knop klikken, kopieert de pagina stilletjes een kwaadaardig commando naar hun clipboard. De "verificatiestap" instrueert hen vervolgens om Win+R te drukken (de Windows Run-dialog), het clipboard te plakken met Ctrl+V en Enter te drukken.

⚠️ Waarom dit zo gevaarlijk is

De gebruiker voert zelf de aanval uit. Er is geen exploit, geen malware download, geen verdachte bijlage. De browser ziet niets fout. Antivirussoftware slaat geen alarm. De aanvaller hoeft alleen maar te wachten.

Deze techniek wordt al actief gebruikt door cybercriminelen wereldwijd — onder andere bij aanvallen op de zorgsector, financiële instellingen en overheden. Juist daarom besloot ik bij jouw organisatie om onze medewerkers hiertegen te trainen — niet met een e-learning, maar met een échte simulatie.

Onze aanpak

De simulatie — veilig maar realistisch

In deze gids laat ik zien hoe ik een volledig werkende ClickFix simulatie bouwde die er van buitenaf ononderscheidbaar is van een echte aanval — met één cruciaal verschil: de payload is volledig veilig. Als een gebruiker de stappen uitvoert, belandt hij op een awareness pagina die uitlegt wat er net is gebeurd.

Waarom niet gewoon GoPhish gebruiken?

GoPhish is een geweldig open source platform voor phishing simulaties — maar het heeft geen ingebouwde ondersteuning voor ClickFix of FakeCaptcha aanvallen. GoPhish is primair gebouwd voor klassieke phishing: een overtuigende e-mail sturen, een neppe loginpagina tonen en credentials harvesten.

ClickFix werkt fundamenteel anders. Er worden geen credentials gestolen — de gebruiker wordt misleid om zelf een commando uit te voeren. Dat vereist:

Wat GoPhish mist voor ClickFix

• Geen clipboard API — GoPhish kan geen JavaScript uitvoeren dat stilletjes iets naar het clipboard kopieert
• Geen multi-page flow — GoPhish serveert één landing page, maar ClickFix heeft meerdere stappen nodig
• Geen Win+R tracking — GoPhish weet niet of een gebruiker daadwerkelijk een commando heeft uitgevoerd buiten de browser
• Geen commerciële alternatieven — tools als KnowBe4 en Proofpoint hebben nog geen kant-en-klare ClickFix templates

De oplossing: GoPhish gebruiken voor wat het goed doet (e-mail delivery en RID tracking) en de ClickFix simulatie zelf bouwen als externe webpagina. GoPhish stuurt de gebruiker naar mijn pagina, mijn pagina doet de rest — en via Sentinel koppel ik alles weer samen.

"Een website vraagt je nooit om Win+R te drukken en iets te plakken. Nooit. Als dat gebeurt, sluit de pagina en meld het bij IT Security."

Stap 1 — E-mail via GoPhish

Medewerkers ontvangen een phishing mail met een link naar de simulatiepagina. GoPhish registreert automatisch wie de mail opent en wie op de link klikt.

Stap 2 — index.html laden

De link opent een neppe Microsoft beveiligingspagina. Het bezoek wordt geregistreerd in Azure AppServiceHTTPLogs met het unieke RID (Recipient ID) van GoPhish.

Stap 3 — ClickFix trigger

De gebruiker selecteert "Enable Passkey Method" en klikt Next. De awareness URL wordt stil naar het clipboard gekopieerd. Een modal instrueert Win+R → Ctrl+V → Enter.

Stap 4 — Awareness pagina

Als de gebruiker de stappen uitvoert, opent de awareness pagina in de Run-dialog. Het bezoek aan /source/content.webp bevestigt 100% dat de gebruiker Win+R heeft uitgevoerd.

Frontend

De webpagina's

De simulatiepagina's zijn zo gebouwd dat ze er identiek uitzien aan echte Microsoft beveiligingsprompts. Elke pagina geeft het GoPhish RID door via URL parameters zodat tracking door de hele keten werkt.

index.html — Eerste contact

De eerste pagina die gebruikers zien na het klikken op de phishing link. Gestyled als een Microsoft account beveiligingspagina met jouw organisatie logo en .onmicrosoft.com domein indicator.

JavaScript// Read ?rid= from GoPhish URL
const params = new URLSearchParams(window.location.search);
const rid = params.get('rid') || 'unknown';

// Pass rid forward to add.html in same directory
document.getElementById('nextBtn').href = `add.html?rid=${rid}`;

add.html — De ClickFix trigger

De kern van de simulatie. Gestyled als een Microsoft MFA setup pagina met radio button opties. Wanneer de gebruiker "Enable Passkey Method" selecteert en op Next klikt:

JavaScriptconst payload = `https://YOUR_AWARENESS_DOMAIN/index.html?rid=${rid}`;

function handleNext() {
  const selected = document.querySelector('.option-tile.selected');
  if (!selected) {
    verifyMessage.style.display = 'block';
    return;
  }
  // Silently copy awareness URL to clipboard
  navigator.clipboard.writeText(payload).then(() => {
    modal.style.display = 'flex'; // Show Win+R instructions
  });
}
ℹ️ Geen echt commando

De clipboard payload is uitsluitend de URL naar de awareness pagina. Er wordt op geen enkel moment een echt commando of script naar het clipboard gekopieerd. De simulatie is volledig veilig.

Tracking via RID

GoPhish maakt voor elke ontvanger een uniek Recipient ID (RID) aan. Dit RID wordt via {{.RId}} in de e-mail link ingevoegd en door alle pagina's heen doorgegeven als URL parameter.

GoPhish template# Landing page URL in GoPhish campaign:
https://YOUR_SIMULATION_DOMAIN/?rid={{.RId}}

# GoPhish vervangt {{.RId}} automatisch, bijv:
https://YOUR_SIMULATION_DOMAIN/?rid=BoziZZb
GoPhish

GoPhish integratie

GoPhish is een open source phishing simulatieplatform. Het verzorgt de e-mail delivery, RID generatie en basis click tracking. Via de REST API halen ik alle campaign resultaten op voor verwerking naar Sentinel.

🔑 GoPhish API authenticatie

GoPhish verwacht de API key als query parameter, niet als Authorization header. Let hierop bij het bouwen van integraties.

Python# Haal alle campaign resultaten op via GoPhish API
resp = requests.get(
    f"{GOPHISH_URL}/api/campaigns/{campaign_id}/results?api_key={GOPHISH_API_KEY}",
    verify=False  # Self-signed cert op localhost
)

# Response bevat per ontvanger:
# id (RID), email, first_name, last_name, timeline

Let op: GoPhish draait op HTTPS ook lokaal — gebruik https://localhost:3333 als GOPHISH_URL, niet HTTP.

Microsoft Sentinel

Data pipeline naar Sentinel

Het hart van de oplossing is een Python script dat elk uur draait op de GoPhish server als cron job. Het combineert GoPhish campaign data met Azure AppServiceHTTPLogs en pusht alles naar een custom tabel in Microsoft Sentinel.

Python sync script

Het script draait elk uur en voert de volgende stappen uit:

Python — sentinel.pyimport json, logging, requests
from datetime import datetime, timezone, timedelta
from azure.identity import ClientSecretCredential
from azure.monitor.query import LogsQueryClient, LogsQueryStatus

# ── Configuratie ──────────────────────────
GOPHISH_URL      = "https://localhost:3333"
GOPHISH_API_KEY  = "YOUR_GOPHISH_API_KEY"
TENANT_ID        = "YOUR_TENANT_ID"
CLIENT_ID        = "YOUR_CLIENT_ID"
CLIENT_SECRET    = "YOUR_CLIENT_SECRET"
DCE_ENDPOINT     = "https://YOUR_DCE_ENDPOINT.ingest.monitor.azure.com"
DCR_IMMUTABLE_ID = "YOUR_DCR_IMMUTABLE_ID"
STREAM_NAME      = "Custom-GoPhish_CL"
WORKSPACE_ID     = "YOUR_WORKSPACE_ID"
LOOKBACK_HOURS   = 2

def push_to_sentinel(credential, records):
    token = credential.get_token("https://monitor.azure.com/.default").token
    url = (
        f"{DCE_ENDPOINT}/dataCollectionRules/{DCR_IMMUTABLE_ID}"
        f"/streams/{STREAM_NAME}?api-version=2023-01-01"
    )
    resp = requests.post(url, data=json.dumps(records),
        headers={"Authorization": f"Bearer {token}", "Content-Type": "application/json"})
    return resp.status_code  # 204 = success

KQL & Dashboard

GoPhishEnriched — de KQL functie

De slimste onderdeel van de oplossing is de GoPhishEnriched KQL functie in Sentinel. Deze combineert GoPhish_CL met live AppServiceHTTPLogs en dedupliceert automatisch zodat elke gebruiker maar één keer voorkomt — ook al heeft het Python script meerdere keren gedraaid.

GoPhishEnriched — Volledige KQL functie

KQL 
// Alleen geldige GoPhish RIDs (bescherming tegen scanners)
let ValidRIDs = GoPhish_CL
| summarize arg_max(TimeGenerated, *) by RID
| distinct RID;

// Stap 1 — Link geklikt (index pagina)
let ClickedLink = AppServiceHTTPLogs
| where CsHost == "YOUR_SIMULATION_DOMAIN"
| where CsUriStem == "/" or CsUriStem == "/index.html"
| extend RID = extract("rid=([^&\" ]+)", 1, CsUriQuery)
| where isnotempty(RID) and RID != "unknown"
| where RID matches regex "^[A-Za-z0-9]{5,10}$"
| where RID in (ValidRIDs)
| summarize ClickedAt = min(TimeGenerated) by RID;

// Stap 2 — ClickFix pagina bereikt
let ReachedClickFix = AppServiceHTTPLogs
| where CsUriStem == "/add.html"
| extend RID = extract("rid=([^&\" ]+)", 1, CsUriQuery)
| where RID in (ValidRIDs)
| summarize ReachedClickFixAt = min(TimeGenerated) by RID;

// Stap 3 — Win+R uitgevoerd (100% bevestigd)
let ExecutedClickFix = AppServiceHTTPLogs
| where CsHost == "YOUR_AWARENESS_DOMAIN"
| where CsUriStem contains "/source/content.webp"
| extend RID = extract("rid=([^&\" ]+)", 1, Referer)
| where RID in (ValidRIDs)
| summarize ExecutedAt = min(TimeGenerated),
    IPs = make_set(CIp) by RID;

// Stap 4 — Dedupliceer GoPhish_CL
let LatestGoPhish = GoPhish_CL
| summarize arg_max(TimeGenerated, *) by RID;

// Stap 5 — Alles samenvoegen
LatestGoPhish
| join kind=leftouter ClickedLink on $left.RID == $right.RID
| join kind=leftouter ReachedClickFix on $left.RID == $right.RID
| join kind=leftouter ExecutedClickFix on $left.RID == $right.RID
| extend ClickFixExecuted = isnotempty(ExecutedAt)
| project CampaignName, RID, FirstName, LastName, Email,
    ClickedAt, ReachedClickFixAt, ClickFixExecuted, ExecutedAt, IPs
| sort by ClickedAt desc
✓ Scanner bescherming

De ValidRIDs lookup en regex filter ^[A-Za-z0-9]{5,10}$ blokkeren XSS payloads, SSTI injection pogingen en botverkeer volledig. Alleen echte GoPhish ontvangers tellen mee in de resultaten.

De belangrijkste les

ClickFix werkt omdat het gebruik maakt van vertrouwen. Gebruikers vertrouwen Microsoft pagina's. Ze vertrouwen MFA prompts. Ze vertrouwen hun werkgever. Een aanvaller die dat vertrouwen nabootst, hoeft geen exploit te gebruiken — de gebruiker doet het werk.

De enige verdediging is bewustzijn. En bewustzijn bouw je niet met een jaarlijkse e-learning, maar met realistische simulaties die mensen daadwerkelijk laten ervaren hoe een aanval werkt. Precies wat ik hier laat zien.
