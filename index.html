Background

What is a ClickFix attack?

ClickFix — also known as FakeCaptcha — is one of the most dangerous social engineering techniques van dit moment. Niet omdat het technisch geavanceerd is, maar juist omdat het zo simpel werkt.

In a ClickFix attack, an attacker creates a fake verification page that looks completely legitimate — think a Microsoft MFA setup screen, a Cloudflare CAPTCHA, or a browser verification prompt. The page asks the user to perform a "verification step".

What the user does not know: the moment they click the button, the page silently copieslletjes een kwaadaardig commando naar hun clipboard. De "verificatiestap" instrueert hen vervolgens om What the user does not know: the moment they click the button, the page silently copies a malicious command to their clipboard. The "verification step" then instructs them to press Win+R (the Windows Run dialog), paste the clipboard with Ctrl+V and press Enter. te drukken (de Windows Run-dialog), het clipboard te plakken met Ctrl+V en Enter te drukken.

⚠️ Why this is so dangerous

The user executes the attack themselves. There is no exploit, no malware download, no suspicious attachment. The browser sees nothing wrong. Antivirus software raises no alarm. The attacker just has to wait.

This technique is already actively used by cybercriminals worldwide — including in attacks on the healthcare sector, financial institutions and governments. This is exactly the kind of attack you need to train yemployees against — not with an e-learning, but with a real simulation.

Our approach

The simulation — safe but realistic

In this guide I show how I built a fully working ClickFix simulation that is indistinguishable from a real attack from the outside — with one crucial difference: the payload is completely safe. If a user follows the steps, they land on an awareness page that explains what just happened.

Why not just use GoPhish?

GoPhish is a great open source platform for phishing simulations — but it has no built-in support for ClickFix or FakeCaptcha attacks. GoPhish is primarily built for classic phishing: sending a convincing email, showing a fake login page and harvesting credentials.

ClickFix works fundamentally differently. No credentials are stolen — the user is tricked into executing a command themselves. That requires:

What GoPhish lacks for ClickFix

• No clipboard API — GoPhish cannot execute JavaScript that silently copies something to the clipboard
• No multi-page flow — GoPhish serves one landing page, but ClickFix needs multiple steps
• No Win+R tracking — GoPhish has no way of knowing if a user actually executed a command outside the browser
• No commercial alternatives — tools like KnowBe4 and Proofpoint do not yet have ready-made ClickFix templates

The solution: use GoPhish for what it does well (email delivery and RID tracking) and build the ClickFix simulation as an external web page. GoPhish sends the user to my page, my page does the rest — and via Sentinel I tie everything back together.

"Een website vraagt je nooit om Win+R te drukken en iets te plakken. Nooit. Als dat gebeurt, sluit de pagina en meld het bij IT Security."

Step 1 — Email via GoPhish

Employees receive a phishing email with a link to our simulation page. GoPhish automatically registers who opens the email and who clicks the link.

Step 2 — Loading index.html

The link opens a fake Microsoft/your organisation security page. The visit is recorded in Azure AppServiceHTTPLogs with the unique RID (Recipient ID) from GoPhish.

Step 3 — ClickFix trigger

User selects "Enable Passkey Method" and clicks Next. The awareness page URL is silently copied to the clipboard. A modal instructs Win+R → Ctrl+V → Enter.

GoPhish

GoPhish integration

GoPhish is an open source phishing simulation platform. It handles email delivery, RID generation and basic click tracking. Via the REST API I retrieve all campaign results for processing to Sentinel.

🔑 GoPhish API authentication

GoPhish expects the API key as a query parameter, not as an Authorization header. Keep this in mind when building integrations.

Python# Haal alle campaign resultaten op via GoPhish API
resp = requests.get(
    f"{GOPHISH_URL}/api/campaigns/{campaign_id}/results?api_key={GOPHISH_API_KEY}",
    verify=False  # Self-signed cert op localhost
)

# Response bevat per ontvanger:
# id (RID), email, first_name, last_name, timeline

Let op: GoPhish draait op HTTPS ook lokaal — gebruik https://localhost:3333 als GOPHISH_URL, niet HTTP.

Microsoft Sentinel

Data pipeline to Sentinel

The heart of the solution is a Python script that runs every hour on the GoPhish server as a cron job. It combines GoPhish campaign data with Azure AppServiceHTTPLogs and pushes everything to a custom table in Microsoft Sentinel.

Python sync script

The script runs every hour and executes the following steps:

Python — sentinel.pyimport json, logging, requests
from datetime import datetime, timezone, timedelta
from azure.identity import ClientSecretCredential
from azure.monitor.query import LogsQueryClient, LogsQueryStatus

# ── Configuratie ──────────────────────────
GOPHISH_URL      = "https://localhost:3333"
GOPHISH_API_KEY  = "YOUR_GOPHISH_API_KEY"
TENANT_ID        = "YOUR_TENANT_ID"
CLIENT_ID        = "YOUR_CLIENT_ID"
CLIENT_SECRET    = "YOUR_CLIENT_SECRET"
DCE_ENDPOINT     = "https://YOUR_DCE_ENDPOINT.ingest.monitor.azure.com"
DCR_IMMUTABLE_ID = "YOUR_DCR_IMMUTABLE_ID"
STREAM_NAME      = "Custom-GoPhish_CL"
WORKSPACE_ID     = "YOUR_WORKSPACE_ID"
LOOKBACK_HOURS   = 2

def push_to_sentinel(credential, records):
    token = credential.get_token("https://monitor.azure.com/.default").token
    url = (
        f"{DCE_ENDPOINT}/dataCollectionRules/{DCR_IMMUTABLE_ID}"
        f"/streams/{STREAM_NAME}?api-version=2023-01-01"
    )
    resp = requests.post(url, data=json.dumps(records),
        headers={"Authorization": f"Bearer {token}", "Content-Type": "application/json"})
    return resp.status_code  # 204 = success

KQL & Dashboard

GoPhishEnriched — the KQL function

The smartest part of the solution is the The smartest part of the solution is the GoPhishEnriched KQL function in Sentinel. This combines GoPhish_CL with live AppServiceHTTPLogs and automatically deduplicates so each user only appears once — even if the Python script has run multiple times. KQL functie in Sentinel. Deze combineert GoPhish_CL met live AppServiceHTTPLogs en dedupliceert automatisch zodat elke gebruiker maar één keer voorkomt — ook al heeft het Python script meerdere keren gedraaid.

GoPhishEnriched — Full KQL function

KQL 
// Alleen geldige GoPhish RIDs (bescherming tegen scanners)
let ValidRIDs = GoPhish_CL
| summarize arg_max(TimeGenerated, *) by RID
| distinct RID;

// Stap 1 — Link geklikt (index pagina)
let ClickedLink = AppServiceHTTPLogs
| where CsHost == "YOUR_SIMULATION_DOMAIN"
| where CsUriStem == "/" or CsUriStem == "/index.html"
| extend RID = extract("rid=([^&\" ]+)", 1, CsUriQuery)
| where isnotempty(RID) and RID != "unknown"
| where RID matches regex "^[A-Za-z0-9]{5,10}$"
| where RID in (ValidRIDs)
| summarize ClickedAt = min(TimeGenerated) by RID;

// Stap 2 — ClickFix pagina bereikt
let ReachedClickFix = AppServiceHTTPLogs
| where CsUriStem == "/add.html"
| extend RID = extract("rid=([^&\" ]+)", 1, CsUriQuery)
| where RID in (ValidRIDs)
| summarize ReachedClickFixAt = min(TimeGenerated) by RID;

// Stap 3 — Win+R uitgevoerd (100% bevestigd)
let ExecutedClickFix = AppServiceHTTPLogs
| where CsHost == "YOUR_AWARENESS_DOMAIN"
| where CsUriStem contains "/source/content.webp"
| extend RID = extract("rid=([^&\" ]+)", 1, Referer)
| where RID in (ValidRIDs)
| summarize ExecutedAt = min(TimeGenerated),
    IPs = make_set(CIp) by RID;

// Stap 4 — Dedupliceer GoPhish_CL
let LatestGoPhish = GoPhish_CL
| summarize arg_max(TimeGenerated, *) by RID;

// Stap 5 — Alles samenvoegen
LatestGoPhish
| join kind=leftouter ClickedLink on $left.RID == $right.RID
| join kind=leftouter ReachedClickFix on $left.RID == $right.RID
| join kind=leftouter ExecutedClickFix on $left.RID == $right.RID
| extend ClickFixExecuted = isnotempty(ExecutedAt)
| project CampaignName, RID, FirstName, LastName, Email,
    ClickedAt, ReachedClickFixAt, ClickFixExecuted, ExecutedAt, IPs
| sort by ClickedAt desc
✓ Scanner protection

The The ValidRIDs lookup and regex filter ^[A-Za-z0-9]{5,10}$ completely block XSS payloads, SSTI injection attempts and bot traffic. Only real GoPhish recipients count in the results. lookup en regex filter ^[A-Za-z0-9]{5,10}$ blokkeren XSS payloads, SSTI injection pogingen en botverkeer volledig. "A website will never ask you to press Win+R and paste something. If that happens — close the page, and report it."

The key lesson

ClickFix works because it exploits trust. Users trust Microsoft pages. They trust MFA prompts. They trust their employer. An attacker who mimics that trust does not need to use an exploit — the user does the work.

The only defence is awareness. And awareness is not built with an annual e-learning, but with realistic simulations that actually let people experience how an attack works. Exactly what I show here.
